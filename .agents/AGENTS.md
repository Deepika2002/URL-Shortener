# Project Customizations

Before performing any task, writing code, or generating configurations for this project, you MUST read and adhere strictly to the foundational architectural blueprints located in the `.ai/` directory.

Specifically, you must always consult:
1. `.ai/status.md`: For the current development progress, completed phases, and latest requirement changes.
2. `.ai/system-architecture.md`: For microservice boundaries and system design.
3. `.ai/tech-stack.md`: For allowed languages, versions, and environment variables.
4. `.ai/schemas.md`: For database structures (ScyllaDB, ClickHouse, Redis) and API contracts.
5. `.ai/common-rules.md`: For strict development workflows, monorepo scaffolding, and code quality standards.

Never deviate from these constraints without explicit user approval.
